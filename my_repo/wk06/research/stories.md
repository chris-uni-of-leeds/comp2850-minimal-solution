## Story S1: Task ordering

**Underlying need**: Because i only have use of a keyboard, which makes navigation a more ardous process

## Story S2: confirmation feedback
**Situation**: When I'm submitting a form (add/edit/delete task)
**Motivation**: I want the confirmation message dispalyed in the are where the form was submitted
**Outcome**: So it can be found easily when using a screen reader
**Underlying need**: Because it is more difficult to locate a confirmation message at the top of the screen 

**Evidence**: Participant A (notes L5), Participant B (notes L3)
**Inclusion risk**: Screen reader, low visual ability
**Type**: Job story
**WCAG**: 4.1.3 Status Messages (AA)

## Story S3: filter feedback
**Situation**: When I'm applying a filter
**Motivation**: I want a confirmation on pressing apply filter, or an error if no matches found
**Outcome**: So I can confirm if I have made a typo
**Underlying need**: As being unable to see the screen increases the possibility of typos, which can be hard to notice


**Evidence**: Participant A (notes L5), Participant B (notes L3)
**Inclusion risk**: Screen reader, low visual ability
**Type**: Job story
**WCAG**: 4.1.3 Status Messages (AA)

## Story S4: filter feedback
**Situation**: When adding a task
**Motivation**: I want the text at the top of the screen to change from "no tasks yet."
**Outcome**: So I can see how many tasks I currently have
**Underlying need**: The lack of dynamic changing, means it always displays no tasks yet, which confuses me if I have already added tasks, increasing stress that I have used the website incorrectly


**Evidence**: Participant A (notes L5), Participant B (notes L3)
**Inclusion risk**: Cognitive, Low Digital literacy
**Type**: Job story
